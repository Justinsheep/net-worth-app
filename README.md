# 我的總資產

手動記錄台股 / 美股 / 加密貨幣 / 現金外幣 / 負債，自動抓報價、算出總資產與淨值，並畫出資產配置圓餅圖。資料存在瀏覽器本機（IndexedDB），不上傳任何地方。

## 怎麼跑起來

需要先裝 [Node.js](https://nodejs.org)（18 以上）。

```bash
npm install      # 第一次才需要
npm run dev      # 啟動，開瀏覽器到終端機印出的網址（通常 http://localhost:5173）
```

## 自動報價（第 2 階段）

分兩條路，都免費、免金鑰：

- **加密貨幣、匯率**：瀏覽器直接抓，即時。打開 App 就會自動更新（每 60 秒一次，也可按「↻ 重新整理」）。
  - 加密貨幣來源：Binance 公開行情（data-api.binance.vision）
  - 匯率來源：open.er-api.com（USD→TWD）
- **台股、美股**：瀏覽器不能直接抓，改用一支 Node 腳本產生 `public/prices.json`：

  1. 編輯根目錄的 `watchlist.json`，把你要追蹤的代號填進去：
     ```json
     { "tw_stock": ["2330", "0050"], "us_stock": ["AAPL", "NVDA"] }
     ```
  2. 跑：
     ```bash
     npm run prices
     ```
  3. 回瀏覽器重新整理，台股/美股就會顯示「即時」報價。

  之後部署到 GitHub 後，`.github/workflows/prices.yml` 會每 30 分鐘自動跑上面這支腳本，就不用手動了。

## 使用重點

- 新增股票/加密貨幣時，**代號**欄位很重要（拿來抓報價）：台股填 `2330`、美股填 `AAPL`、加密貨幣填 `BTC`。
- 「現價」欄位是後備：抓不到報價時才用。
- 右上 USD/TWD 預設「自動」（用即時匯率）；按一下切「手動」就能自己輸入。
- 底部可匯出 / 匯入 JSON 備份。

## 檔案結構

```
src/
  db.js                  Dexie 資料庫定義
  store.js               唯一的資料存取層（未來換 Supabase 只改這支）
  calc.js                分類、台幣換算、報價解析
  prices.js              前端報價服務（加密貨幣 + 匯率 + 讀 prices.json）
  App.jsx                主畫面
  components/            圓餅圖、表單、列表
scripts/
  fetch-prices.mjs       台股/美股報價腳本
watchlist.json           要追蹤的股票代號
.github/workflows/
  prices.yml             部署後定時更新報價
```

## 成本與報酬率（第 3 階段）

- 新增股票/加密貨幣時，可填「總投入成本」和「買入日期」。成本單價 = 總投入 ÷ 數量。
- 同一檔不同時間買的，就分開新增（代號填一樣），每筆各自算報酬率，下面自動給一個合計。
- 報酬率用**原幣**計算（純看這檔漲跌，排除匯率）；損益金額也用原幣顯示。
- 上方「未實現損益」是所有已填成本部位換算成台幣的加總。
- 沒填成本的部位照常顯示現值，報酬率顯示「—」。賣出（已實現損益）目前不做。
